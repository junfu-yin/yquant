# 04 · 外包工作说明书（SOW）

> yquant 项目文档包 · 编号 04
> 版本 v1.0 · 2026-07-05
> 本文件与《03 技术方案 v2》共同构成外包工作的完整依据：03 定义"做成什么样"，本文件定义"怎么干、怎么交、怎么算完成"。冲突时以 03 为准。

---

## 1. 项目概述与角色

- **项目**：yquant——**美股（主）与港股（辅）**个人投资者的 AI 投研副驾与纪律化交易辅助系统（定位与背景见 01/02/03 篇）。
- **委托人**：提供需求裁决、LLM API key 与（可选）付费行情源 token、评测集终审、里程碑验收。承诺工作日 24 小时内答复阻塞性问题。
- **承接方**：1 名具备以下能力的开发者——Python 工程化（类型注解/测试/打包）、pandas 数据处理、有 backtrader 或同类回测框架实战经验、调用过 LLM API 并做过结构化输出、了解美股/港股基本交易规则（T+1/T+2 交收、LULD/熔断、VCM/CAS、拆股分红调整、PDT）。**加分项**：用过 yfinance/AkShare、对接过 SEC EDGAR/HKEXnews、写过 Streamlit。
- **工作模式**：远程、里程碑制。预估总量 35–46 人日（兼职 6–9 周，全职约 7–9 周含陪跑）。

## 2. 工作包（WP）定义

> 每个 WP 对应 03 篇一个里程碑。**出口判据（Gate）逐条满足才进入下一 WP**；Gate 即验收，无口头验收。

### WP0 · 环境与假设验证（3–4 人日）
**任务**：初始化仓库（03 §6 结构）、Poetry 依赖锁定、CI（ruff+mypy+pytest）、config 体系与密钥注入；实测并落定 03 篇全部 AS 假设（AS-1 公告接口与正文可得性——美股 EDGAR/港股 HKEXnews、AS-2 新闻接口、AS-3 财务公告日字段、AS-4 退市/摘牌股可得数量、AS-5 LLM 成本、AS-6 交收周期/费率/微观结构规则与生效时点核对、备源可用性），产出 `docs/assumptions.md`（每条含：结论、证据脚本、样例数据截图/文件）。
**Gate**：CI 全绿；yfinance/AkShare/EDGAR/HKEXnews 各跑通一次冒烟拉取；assumptions.md 经委托人确认签字（issue 关闭为准）。

### WP1 · 数据基座（6–8 人日）
**任务**：实现 03 §5.1 全部——DataSource 协议与各 adapter（yfinance/AkShare/Stooq/EDGAR/HKEXnews）、DataRepo、增量更新与降级逻辑、`market_rules` 交收/微观结构规则、质检器、全量历史数据首次灌入。
**Gate**：①美股 + 港股 2010 至今日线+拆股/分红调整因子入库，质检通过率 ≥99%（质检报告随附）；②`get_universe("2018-06-01", market="us")` 返回集合包含此后退市/摘牌的股票且数量达 AS-4 落定值；③mock 断源测试证明"重试→切源→跳过并报告"链路；④配置切换主数据源后测试套件仍通过；⑤`market_rules` 单元测试覆盖美股 2024-05-28 T+2→T+1 变更点、PDT 触发、港股 lot_size/VCM/CAS、半日市与临时休市（台风/黑雨）等 case。

### WP2 · 回测引擎（6–8 人日）
**任务**：03 §5.2——美股/港股约束 Broker（含自建交收批次账本、PDT 规则）、成本模型（美股 SEC费+TAF/港股印花税+交易征费）、run_backtest 与报告、walk-forward 评估器；实现 03 §8 陷阱测试集 T0–T7。
**Gate**：**陷阱测试集全绿并纳入 CI（含 T3b 交收周期时变、T6 熔断/波动控制、T7 PDT 规则）**；确定性测试通过；性能达标或提交经委托人认可的、符合 §5.2 性能实现路径的方案说明。

### WP3 · 基准策略库（5–7 人日）
**任务**：03 §5.3——先为 S1/S2/S3 各写一份 spec 文档（规则的每个细节：调仓时点、筛选顺序、边界处理如成分不足 20 只时怎么办）交委托人确认，再实现，再出 walk-forward 报告与结论文字。
**Gate**：三份 spec 获确认；三份报告含成本三档、参数敏感性、与 B0（买入持有标普500/SPY）对照；结论如实（负面结论不影响验收）；随附 K1 判据所需的对照汇总表。

### WP4 · AI 简报引擎（7–9 人日）
**任务**：03 §5.4——采集预过滤、输入裁剪（要素段抽取+左截断+规则前置分类）、llm_client（重试/超时/记账/版本化）、EventCard 结构化与硬校验（数字**归一化后**回验为重中之重，含英文量纲 K/M/B）、分市场班次覆盖窗口（港股17:00/美股06:00补采/07:30隔夜/08:00合并）、分级汇总、评测集（收集 ≥120 条素材+预标注，严重类目加密）与评测脚本。
**Gate**：委托人终审后的评测集上达到 03 §5.4 指标（分类 ≥85% / 严重度 ±1 ≥85% 且 severity≥4 召回 ≥95% / 方向 ≥80% / 未验数字漏杀 =0）；连续 5 个交易日无人工干预产出简报（含美股隔夜补采班次）；日成本账单 ≤¥2（AS-5 实测报告随附）。

### WP5 · 纪律引擎 + UI + 调度（8–10 人日）
**任务**：03 §5.5–5.7 全部。
**Gate**：用户故事 US-1~US-5 逐条录屏演示通过；checklist 强制性 UI 测试通过；风控 10 场景单元测试通过；daemon 连跑 3 个交易日任务全成功（job_runs 为证）；从零部署文档实测 ≤30 分钟（委托人在干净环境复现）。

### WP6 · 陪跑与移交（机动，约 30 天内响应制）
**任务**：真实使用期缺陷修复（不加功能）；移交：架构讲解会 1 次（录屏）、运维手册（常见故障、数据修复、prompt 升级流程）。
**Gate**：陪跑期结束时 open 的 P0/P1 缺陷清零；移交材料齐。

## 3. 缺陷分级与响应

P0（数据错误导致简报/回测结论错误、系统不可启动）：24h 内响应，48h 修复。P1（功能不可用但可绕过）：72h 修复。P2（体验问题）：下个批次。陪跑期内适用；此前以 Gate 为准。

## 4. 交付物清单（最终）

1. 代码仓库（含全部历史提交，squash 禁止）+ 全绿 CI。
2. `docs/`：assumptions.md、三策略 spec 与报告、评测集与评测报告、部署文档、运维手册、本文档包回填的变更记录。
3. 数据资产：首灌的 parquet/SQLite 数据包或可一键重建的脚本（二选一，重建耗时须 <4 小时）。
4. 演示材料：每个 Gate 的验收录屏。

## 5. 协作与变更流程

- **看板**：GitHub Projects，卡片粒度 ≤1 人日；每卡关联 WP 编号。
- **PR 规则**：一 PR 一主题；描述含"对应 03 篇章节 + 自测证据"；委托人或其代理 review 后合入 main；main 始终可部署。
- **周报**：每周五简报（本周完成/下周计划/阻塞/假设变化），异步文字即可。
- **答疑**：疑问先查 03 §12 FAQ → 查 02 篇相关实验 → 仍无解则开 issue 打 `question` 标签；委托人 24h 内答复。**口头/聊天软件达成的任何决定 24h 内必须回填 issue，否则视为无效。**
- **变更管理**：任何偏离 03 篇规格的实现（包括"更好的做法"）走变更单：issue 说明动机/影响/工时变化 → 委托人书面同意 → 更新 03 篇变更记录与 02 篇 ADR。未走流程的偏离在验收时按缺陷处理。
- **范围保护**：03 §2.4 非目标清单内的任何功能请求（无论来自谁）一律进 `docs/icebox.md`，不进开发队列。

## 6. 验收与付款建议

按 Gate 分期：WP0 完成付 10%、WP1–WP2 各 15%、WP3 10%、WP4 20%、WP5 20%、WP6 移交后付尾款 10%。（比例为建议值，双方合同为准。）**"演示能跑"不构成验收；"Gate 判据逐条满足且证据归档"构成验收。**

## 7. 知识产权与保密

交付代码及文档知识产权归委托人；承接方不得留存或复用委托人的密钥、交易日志与持仓数据；开源依赖遵循其原始许可证（backtrader GPLv3 影响评估：本项目自用不分发，无传染问题；若未来开源需另行评估——已记入 icebox）。
