## 实验 0：先问最根本的问题——这个项目的"有用"到底指什么？

拿到 v1 方案时我的第一反应不是评审技术选型，而是发现方案缺少一个东西：**成功的定义**。方案写了做什么（数据→分析→决策→执行），没写做到什么程度算成功、对谁成功。

我强迫自己给出可证伪的定义。候选答案有三个：

- 定义 A："系统能自动赚钱"——年化收益跑赢基准。
- 定义 B："系统是个完整的量化平台"——功能齐全、架构漂亮。
- 定义 C："系统持续改善使用者（一个美股/港股个人投资者）的决策质量、风险控制和时间效率，且使用者愿意长期用它管真钱。"

定义 B 首先排除——这正是委托人警告的"做得完美但没有用"。功能齐全从来不是价值，是成本。

定义 A 很诱人，但我在实验 1 里会论证它对个人而言大概率不可达，把它当北极星会把整个项目引向悬崖。

所以采用定义 C，并把它拆成三个可度量的代理指标（详见 03 §2.3）：**决策质量**（重要信息的遗漏率、操作与既定规则的偏离率）、**风险控制**（回撤是否被约束在预设范围）、**时间效率**（每日投研时间从小时级降到分钟级）。外加一个终极检验：**30 天后使用者是否还在每天打开它**——工具的留存率是"有用"最诚实的裁判。

## 实验 1：核心假设的证伪尝试——"个人 + ML/LLM 预测价格能赚钱"成立吗？

v1 方案的隐含世界观是：数据 + 模型 → 信号 → 收益。Phase 3 甚至规划了"AI 自主执行"。我对这个世界观做压力测试，站在反方尽全力攻击它：

**攻击 1：对手盘结构。** 美股/港股量化赛道上的对手是配备全市场 tick 数据、另类数据、数百研究员和微秒级执行的机构（美股尤甚，是全球流动性最深、机构化最彻底的市场）。个人用免费日线数据 + 通用 LLM 参与同一个零和游戏（扣除交易成本后是负和），凭什么赢？唯一诚实的回答：凭机构看不上的容量、更长的持有周期、以及不受考核约束的耐心——注意，这三样优势**没有一样来自"预测得更准"**。

**攻击 2：信息时效链路。** v1 把"LLM 新闻情绪分析"作为核心切入。做个思想实验：一条利好新闻发布 → 爬虫抓到（分钟级延迟）→ LLM 打分（秒级）→ 交易时价格早已反映（美股/港股虽允许日内交易，但盘后 8-K/业绩公告的价格跳空在开盘瞬间完成）。而机构的事件驱动系统在**秒级/毫秒级**完成同样动作，价格在个人能交易之前早已消化。结论：**新闻情绪作为短线交易信号，对个人是负期望的**——买到的全是已经涨过的。它剩下的价值是什么？见实验 3。

**攻击 3：回测的自我欺骗机制。** 免费数据 + 默认回测框架的组合几乎必然产出虚假的好结果：不含退市/摘牌股（幸存者偏差，美股每年因并购/退市摘牌数百家，港股亦有大量仙股停牌摘牌）、用了公告日之前不可知的财务数据（前视偏差）、忽略停牌/熔断/LULD 期间买不进卖不出（成交偏差）、低估费率与滑点（成本偏差）。个人没有风控团队帮忙挑错，**回测系统骗人比骗过市场容易一万倍**。

**攻击 4：Phase 3"AI 自主执行"的风险不对称。** LLM 会产生幻觉、会被 prompt 注入、会在分布外场景失控；自动执行意味着把这些尾部风险直接接到银行账户上。同时 2024–2025 年程序化交易监管趋严，个人自动化下单的合规路径本就狭窄。收益上限是"省去手动点几下"，损失下限是本金——这个赌局的赔率荒谬。

**反方获胜。** 判决：v1 的"预测→信号→自动交易"主线不成立，必须重构。但注意，判决书里同时留下了三条生路——个人的真实优势（小容量、长周期、耐心）、LLM 的另一种用法（实验 3）、以及回测的另一种用途（实验 5）。

## 实验 2：换个问法——个人投资者真正稀缺的是什么？

如果"更准的预测"不可得，那系统应该供给什么？我列出个人投资者的典型亏损来源，按可干预性排序：

1. **信息处理带宽不足**：自选股 20 只，每天公告、新闻、财务动态几十条，根本读不完；重大利空（内部人大额减持、并购重组、监管调查/SEC 问询、集体诉讼、盈利预警）常常事后才知道。→ **完全可干预**，这恰是 LLM 的主场。
2. **行为偏差**：追涨杀跌、拿不住、亏损加仓、报复性交易。大量研究表明散户亏损的主因是行为而非选股。→ **可部分干预**，用工程手段（checklist、冷静期、强制日志）对抗人性。
3. **验证工具缺失**：听来的策略/自己的想法，没有可信手段检验，只能拿真钱试错。→ **可干预**，前提是回测系统自身可信（实验 5）。
4. **信息与执行速度劣势**：→ **不可干预**，绕开它（不做短线、不做事件套利）。
5. **资金与工具劣势**（无法做空、无低成本杠杆）：→ **不可干预**，绕开。

结论：系统的供给应该对准 1、2、3。这就是 v2 三支柱的由来：**AI 投研简报（供给带宽）、纪律引擎（供给自制力）、可信回测（供给验证能力）**。

## 实验 3：LLM 在这个系统里的正确角色是什么？

死胡同记录：我最初顺着 v1 思路设计了"LLM 情绪因子"——新闻→情绪分→合成因子→进回测。推演到回测环节卡死：要回测情绪因子，需要**历史新闻的精确时间戳全集**（否则前视偏差），免费渠道拿不到高质量的历史新闻存档；即便拿到，实验 1 攻击 2 已证明该因子对个人交易者时效上不可用。**此路不通，放弃将其作为 v1 核心**（保留为 Phase 2 的实验性选修）。

那 LLM 还剩什么？我列出 LLM 相对人类使用者的真实比较优势：**不知疲倦地读、结构化地写、不带情绪地对照规则**。对应三个角色：

- **初级研究员**：每天读完自选股的所有公告/新闻，输出结构化事件卡（类型/严重度/利多利空/一句话摘要/原文链接）。不预测涨跌，只回答"发生了什么、按什么优先级看"。
- **合规官**：交易前对照 checklist 提问（符合策略规则吗？在冷静期吗？超单票上限吗？），交易后强制留痕。
- **复盘教练**：每周把交易日志和当初的理由对照实际结果，指出"计划外操作"和情绪化模式。

关键设计约束（从 ai-hedge-fund 的 deterministic constraints 设计得到印证）：**硬约束永远由规则代码执行，LLM 只在规则划定的空间内做软判断**。LLM 的每个输出必须可溯源（附原文链接）、可校验（JSON schema + 数字回验）、可评测（人工标注集）。一句话：**把 LLM 从"预言家"降级为"可审计的研究员"，它反而变得可靠且有用。**

## 实验 4：定位三选一的对决

把候选定位摆上台面互搏：

- **定位 A：全自动 AI 交易机器人**（v1 的终局）。实验 1 已判死刑：期望收益存疑、尾部风险巨大、合规路径窄、且市面上 Qbot 等先例证明"大而全"的下场。
- **定位 B：轻量版 Qlib（研究平台）**。技术上可行，但价值存疑：研究平台的产出是论文式结论，不回答"我今天该干什么"；且与 Qlib 正面竞争毫无胜算。委托人要的是"有用"，不是"又一个框架"。
- **定位 C：AI 投研副驾 + 纪律引擎**。每天真实使用、直接对准实验 2 的三大稀缺品、生态位无人占据（01 篇 §3 的空缺区）、失败模式温和（最差情况 = 一个自动读公告的工具，仍然有用）。

打分（价值确定性 / 技术可行性 / 差异化 / 风险）：C 全面胜出。**选 C。** 但保留 A 的一个远期光点：若 C 运转一年、纪律层数据充分，"半自动执行"可以作为远期选项重新评估——写入 roadmap 的 icebox，不写入承诺。

## 实验 5：回测系统的哲学定位——寻宝图还是测谎仪？

流行的用法把回测当寻宝图：扫参数、找到年化 40% 的曲线、上真钱。这条路的终点是过拟合献祭。我给 v2 的回测系统定下相反的哲学：**回测是测谎仪**——它的首要任务是把"看起来能赚钱其实不能"的想法在花真钱之前杀死。

由此推出三条设计铁律：

1. **先测系统自己有没有撒谎**：内置"回测陷阱测试集"（前视/幸存者/交收周期/停牌与熔断/成本/复权六类，03 §8），全绿才有资格谈策略。
2. **样本外优先**：一切结论以 walk-forward 样本外为准，样本内结果仅供参考；参数敏感性必须报告（好策略应该对参数不敏感）。
3. **诚实基准**：所有策略与"买入持有标普500（港股策略以恒生指数对照）"对照；跑不赢就明说跑不赢——**"证明某想法无效"同样是系统的正向产出**，它避免了真金白银的学费。

策略库因此选择低频、规则简单、学术与实务长期验证过、容量对个人无约束的三类作为基准（双均线趋势择时 / 红利低波轮动 / 动量轮动），拒绝一切需要精细参数调优才能看的策略。

## 实验 6：预演失败（premortem）——一年后项目死了，验尸报告怎么写？

强迫自己写出五种最可能的死法，并为每种预置解药：

1. **死法一：数据接口天天挂，修接口修到心态爆炸。** yfinance/AkShare 封装的上游接口改版是常态。→ 解药：适配器模式 + 多备源（yfinance/AkShare/Stooq + 公告走 SEC EDGAR/HKEXnews 官方一手）+ 本地落盘为唯一真实数据源 + 数据质检日报（03 §5.1 M1 模块的硬性要求）。
2. **死法二：AI 简报沦为没人看的日经噪音。** 每天 20 条"中性"事件，两周后使用者不再打开。→ 解药：严格分级（severity≥4 才推送）、"今日必看 Top3"机制、以及 kill 判据 K3（连续两周使用<2 次即停下重新设计形态，而不是硬加功能）。
3. **死法三：回测跑出漂亮曲线，实盘立刻打脸。** → 解药：实验 5 的三条铁律 + 强制成本三档敏感性报告（0/1x/2x）。
4. **死法四：范围蔓延，六个月还在写框架。** v1 的 scope（舆情爬虫、大类资产、自主交易……）已现端倪。→ 解药：v2 明确"非目标"清单（03 §2.4），任何新想法先进 icebox，每月只在里程碑评审时讨论一次。
5. **死法五：外包交付物看着能跑，实际不可维护/不可信。** → 解药：验收标准全部写成可执行测试（03 §8、04 SOW），"演示跑通"不算验收，"测试集全绿 + 文档齐"才算。

## 实验 7：为什么这些结论需要对照 GitHub 已有工作才能成立？

自我审查：以上推理会不会是闭门造车？01 篇的外部证据提供了三重校验：

- ai-hedge-fund（60.8k star）与 TradingAgents-CN（13k star）双双在 README 中声明**仅供教育研究、不做/不提供实盘指令**——两个最强的 LLM 交易项目都不敢跨过实盘线，与实验 1 的判决相互印证。
- TradingAgents/TradingAgents-CN 已把"LLM 深度分析单只股票（原生支持美股，CN 版扩展 A 股/港股）"做到可用，说明**该能力本身不再稀缺**；稀缺的是组合级日常流、可验证性、纪律闭环——与实验 2/3 的推导吻合，这就是 v2 的差异化落点。
- Qbot 的"大而全但难落地"与 backtrader 的"无交收/停牌/熔断等市场约束内置"分别印证了实验 6 的死法四与死法三。

内外证据闭环，定位成立。
