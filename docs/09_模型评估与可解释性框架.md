# 09 · 模型评估与可解释性框架（含黑箱模型治理）

> yquant 阶段二文档 · 编号 09 · v1.0 · 2026-07-05
> 依赖：07（评估数据全部来自事件账本与 manifest，天然 point-in-time）、08（治理阶梯复用上线阶梯）。
> 立场（ADR-14）：**不禁止黑箱，禁止不可度量的黑箱。** 对 NN 这类模型，系统不承诺"解释它为什么这样想"（做不到且易自欺），承诺的是四件更诚实的事：它见过什么输入、此刻多有把握、今天像不像它的训练世界、它在历史同类条件下表现如何。委托人的原话"系统应该可以告诉整个黑箱的表现到底如何"——本篇就是这句话的工程化。

---

## 1. 统一抽象：一切信号来源皆 SignalProvider

规则策略（核心 C1–C3、卫星 S-A）、LLM 事件研判与评分（卫星 S-B/S-C）、未来的 NN 打分器，一律实现同一协议、走同一评估与治理流程——**评估框架对模型类型无差别，这样黑箱进来时无需新造轮子**：

```python
class SignalProvider(Protocol):
    provider_id: str                      # 含语义版本，如 "nn_daily_scorer@0.3.1"
    def predict(self, as_of: date, universe: list[str],
                repo: DataRepo) -> list[Inference]: ...
    def model_card(self) -> ModelCard: ...

class Inference(BaseModel):
    symbol: str
    output: float | Literal["buy","sell","hold","abstain"]
    confidence: float            # [0,1]，必填；不会估不确定性的模型没有上线资格
    explain: ExplainContract     # §4
class ModelCard(BaseModel):      # 注册时必填，入库并展示于 UI
    purpose: str; inputs: list[str]; training_window: tuple[date,date] | None
    knowledge_cutoff: date | None    # v3(ADR-24): LLM/ML provider 必填(kind∈{llm,ml_blackbox});
                                     # 规则策略可为 None。缺失则评估流水线拒绝准入(06 §7)
    known_limits: list[str]; owner: str; risks: list[str]; eval_report_ref: str
```

## 2. 离线评估（准入第一关）

全部基于 manifest 的 as-of 读取（07 §3），杜绝前视；对照三条基线：B0 买入持有、单因子动量、随机打分（随机基线赢不了就没资格谈下一步）。

- **打分型**（输出连续分）：滚动 RankIC（月窗）及其 95% 置信区间、按分数十分位的多空组合价差、信号衰减曲线（预测第 1/5/20 日的 IC 衰减）、隐含换手与成本后净价差。
- **分类型**（输出方向/事件判断）：混淆矩阵、PR 曲线、**校准**（可靠性图 + Brier 分数——模型说 70% 把握时是否真有 70% 对）、弃权率与"弃权换准确率"曲线。
- **通过线（默认，config 可调但改动留痕）**：成本后十分位价差 >0 且滚动 RankIC 95%CI 下界 >0 于 ≥60% 的滚动窗口；或分类校准误差（ECE）<0.08 且弃权后准确率 ≥85%。达不到 → 打回，不进入影子。

**J3 污染约束（v3，ADR-24，一等公民约束）**：`kind∈{llm,ml_blackbox}` 的 provider（如卫星层 S-B/S-C）的离线评估**只承认 ModelCard 声明的 `knowledge_cutoff` 之后的样本**：
- 评估流水线按 `knowledge_cutoff` 切分回测区间，截止**之后**为有效前向样本 → 计入上述通过线；截止**之前**的样本一律打 `contaminated` 标记，**仅供展示与探索，禁止计入任何通过/晋级判据**。
- **规则策略（C1–C3/S-A）不受此约束**，可用全历史回测 + walk-forward。
- ModelCard 缺失 `knowledge_cutoff` 的 LLM/ML provider **拒绝准入**（视同不完整）。CI 断言由 06 §7 落地。
- 诚实备案：S-B/S-C 的"历史高夏普"引自论文（Booth / Lopez-Lira），**不构成我们的预期**；我们的预期在前向数据（影子 ≥40 交易日 + 模拟盘 ≥60 交易日）出来之前为"未知"。

## 3. 黑箱刻画四件套（运行期常驻）

1. **表现仪表**：滚动 IC/命中率/成本后贡献，并按市场状态分桶展示（牛/熊/震荡 × 波动率高低分位，状态由指数 200 日均线与波动分位规则化定义）——回答"它在什么天气下行、什么天气不行"。
2. **归因面板**：permutation importance 为主、SHAP 为辅，输出 top 特征贡献；面板固定角标标注 **"描述性归因，非因果解释"**——归因说明模型看重什么，不说明世界为何如此。
3. **分布哨兵（OOD/漂移）**：特征级 PSI 月检（>0.2 告警，>0.3 冻结候选）；单次推理级 OOD 分数（对训练分布的马氏距离或密度比，超阈 → 该笔强制 `abstain` 并入账）；标签漂移（滚动窗口内实际收益分布 vs 训练期）。**今天不像它的训练世界，它今天就没有发言权。**
4. **行为测试（金融版 CheckList）**：金标准回归集（golden 数据上冻结的输入→输出对，变更后逐条比对）+ 不变性检验（对无关扰动如股票代码重排、微小噪声，输出应稳定）+ 方向性预期（构造教科书场景，如"业绩预增+低估值"合成样本，输出方向不应荒谬）。行为测试进 CI，与 06 §8 门禁同级。

## 4. 解释契约（每个信号的强制随附对象）

```python
class ExplainContract(BaseModel):
    kind: Literal["rule","llm","ml_blackbox"]
    confidence: float
    ood_score: float | None          # ml_blackbox 必填
    regime_tag: str                  # 当前市场状态桶
    evidence: list[str]              # rule→规则条文引用；llm→原文链接+rationale；
                                     # ml_blackbox→top归因特征及其当前取值
    similar_history: list[str]       # 账本中历史相似情形的 event_id（近邻检索），
                                     # 附各自后续结果 —— "上次它这么说，后来怎样"
    caveats: list[str]               # 强制至少一条，如"OOD偏高"/"该状态桶历史样本<30"
```

UI 上任何建议单展开即见契约全文；`similar_history` 可点击回放。**没有契约的推理结果，账本拒收**（07 §2 信封校验联动）。

## 5. 漂移响应阶梯与退役

观察（S3 记录）→ 告警（PSI>0.2 或滚动 IC CI 触零，S2）→ **冻结**（provider 暂停参与 proposal，自动回落到规则基线运行，事件入账）→ 复核（离线重评 + 行为测试回归）→ 恢复或**退役**。退役硬条件（满足其一）：滚动 12 窗 RankIC CI 连续 6 窗含 0；月度 OOD 强制弃权率 >30%；引发 P0 事故且根因在模型本身。退役写讣告：ModelCard 归档 + 生命周期总结报告（供后来者避坑）。

## 6. 准入治理阶梯（复用 08 阶梯，对"新 provider"逐个适用）

注册（ModelCard+离线报告）→ 评审 → **影子 ≥40 交易日**（只记 inference 不参与 proposal，期间四件套面板必须已可用）→ **模拟盘 ≥60 交易日**（参与虚拟组合，与冠军并跑）→ 试点期资金参与上限：**该 provider 驱动的仓位 ≤ 组合 10%**，此上限的放宽属于变更事项。冠军-挑战者：任何"替换现有信号源"的诉求，一律以挑战者身份影子并跑 ≥40 日、凭账本数据评审，禁止拍脑袋切换。

## 7. 走查示例：假设引入"NN 日频打分器 v0.1"

① 注册：ModelCard 声明输入（量价+基本面 32 特征）、训练窗（2016–2023）、**知识截止日期 2023-12-31**、已知局限（未见 2024 后 AI 主导的大盘风格）；② 离线：manifest 化重放**知识截止后**的 2024–2026 样本外（截止前区间标 `contaminated` 不计入判定，J3/ADR-24），RankIC CI 下界 0.4%、成本后十分位价差年化 +2.1%、随机基线通过——达线；③ 影子 40 日：仪表显示震荡桶 IC 显著、趋势桶归零，OOD 弃权率 6%；④ 模拟盘 60 日：与冠军（规则卫星 S-A）并跑，成本后贡献 +0.8% vs +0.5%，但一次 PSI 告警（板块权重再平衡致市值/风格特征漂移）触发冻结 5 日、复核后恢复——全程账本可查；⑤ 评审决定：以 10% 仓位上限进入试点，区间书增补该 provider 条目。**任何一步不达标，流程自然终止，系统始终能回答"它到底表现如何"。**

## 8. 验收（对应 WP10）

核心 C1–C3、卫星 S-A 与 LLM 评分 S-B/S-C 完成 provider 化改造（含 ModelCard 与契约输出，LLM 型必填 `knowledge_cutoff`）；离线评估流水线在 golden 数据出三份基线报告（含 J3 污染切分）；四件套面板演示（用 C1 的参数扰动变体扮演"挑战者"完成一次冠军-挑战者影子对比）；OOD 强制弃权与冻结阶梯在注入漂移的合成数据上演示；行为测试入 CI。
