## 1. 新增工作包

### WP7 · 可观测与回放地基（7–9 人日）｜规格：07 篇
账本 DDL 与写入层（信封强校验、幂等键）、manifest 机制与勘误流程、run digest、replay/asof/incident-collect 三个 CLI、告警框架、事故五步法工具化、T11/T13。
**Gate**：07 §8 全项，含合成缺陷上的取证演练报告。
**排期强约束**：**必须紧随 WP2 之后、先于 WP3–WP5 执行**。理由：信封与账本对各业务模块是侵入性架构约束，后补等于全量重构；策略/简报/纪律模块从第一行代码起就应"生在账本里"。

### WP8 · 测试体系与金标准数据（5–7 人日）｜规格：06 篇
P1–P9 指标脚本化与健康面板、属性测试（双 Broker ≥10 万随机用例）、T8–T12、golden 数据集制作与哈希入库、混沌注入脚本、CI 矩阵与发布门禁、迁移脚本机制。
**Gate**：06 §9 全项（T7 除外，归 WP9）。

### WP9 · PaperBroker 与上线阶梯流水线（7–9 人日）｜规格：08 篇
共享约束层重构（回测/模拟同源）、PaperBroker 全生命周期（撮合/估值/拆股与分红调整/对账/冻结联动）、shadow 与 paper 两种模式管道、T7 双引擎等价、孪生对照报表、执行回填与滑点闭环（为 L3 备）。
**Gate**：T7 全绿；L1 影子 20 交易日达标报告；L2 首月（可用 golden 数据加速模拟）对账零差异；拒单/降级场景演示。

### WP10 · 模型评估与可解释性框架（6–8 人日）｜规格：09 篇
Provider 协议与现有四个信号源改造、ModelCard 与解释契约（含账本联动校验）、离线评估流水线与三基线、四件套面板、OOD 与冻结阶梯、行为测试入 CI、冠军-挑战者流程工具化。
**Gate**：09 §8 全项。

### WP11 · 上线运营与演练（3–4 人日）｜规格：08 §4/§6、06 §6
运行手册全文（告警条目↔runbook 段落一一对应）、《预期区间书》模板+首份实例（核心层 C1–C3/卫星 S-A 给区间，S-B/S-C 标"观察中"）、消防演习×1 与混沌演练×1（另一次在陪跑期由委托人主持）、L2→L3 Gate 评审材料包模板。
**Gate**：08 §7 WP11 项；委托人按手册独立完成一次日检+一次告警处置（录屏为证）。

### WP12 · 风险引擎（M8）+ 核心层策略（7–9 人日）｜规格：03 §5.8/§5.3、06 §3
M8 事前仓位控制四子机制（`risk/`：`engine.py`(apply_risk_controls)/`vol_target.py`/`crowding.py`/`circuit_breaker.py`）、核心层 C1 全球多资产双动量 + C2 趋势闸门 + C3 波动率目标（`strategies/core/`）、`risk_events` 表写入与回放、集中度快照周更、陷阱测试 T14/T15。
**Gate**：T14/T15 全绿；06 §2 P10 触发正确性达标；核心层 walk-forward 报告交付且结论诚实（K1 判定材料，03 §9）；`risk_events` 可 `replay --strict` 复现。
**排期强约束**：**M8 须先于任何卫星层 provider 参与 proposal**——事前仓位控制是卫星层进入组合的前置护栏（J4/ADR-25）。

### WP13 · LLM 双策略 provider 化（S-B/S-C）（8–10 人日）｜规格：03 §5.3/§5.4、09 §2、06 §7
S-B（LLM 财报方向评分，Booth 复制，帽 10%）与 S-C（LLM 新闻漂移评分，Lopez-Lira 复制，帽 5%）实现为 `SignalProvider`（`brief/signal_provider.py`），产**评分不产订单**（ADR-22）；ModelCard 含 `knowledge_cutoff` 必填；离线评估遵守 J3 污染约束（仅承认知识截止后样本，截止前标 `contaminated`，06 §7 CI 断言）；接入 09 治理阶梯（注册→离线→影子→模拟盘→仓位帽）。因 token 解限（ADR-27），评测集规模与打分频率可放大。
**Gate**：09 §8 对 S-B/S-C 项；污染标记断言在 CI 生效；两 provider 完成影子期数据积累演示（不要求前向达标——去留由前向裁决，非本包 Gate）。

## 2. 与阶段一工作包的整体排期（修订版单线程顺序）

WP0 → WP1 → WP2 → **WP7** → WP3 → WP8 →（WP4 ∥ WP9，二者无依赖可并行；单人则先 WP9）→ WP5 → WP10 → **WP12（M8+核心层）→ WP13（LLM 双策略 provider 化，依赖 WP10 的 provider 框架与 WP12 的 M8 护栏）** → WP11 → **L1 影子（≥20 交易日，期间仅修缺陷）→ L2 模拟盘（≥60 交易日）→ Gate 评审 → L3 试点**。

阶段二新增工时合计 **43–56 人日**（原 WP7–WP11 计 28–37，新增 WP12 7–9 + WP13 8–10）；两阶段合计约 78–102 人日。L1/L2 为日历时间（约 4 个月），与收尾开发部分重叠，人力投入为响应制。

## 3. 验收证据的统一形态

自本增补起，一切 Gate 证据以**账本导出包**为准（事件链 + manifest + digest + 相关报告），录屏仅作辅助。委托人验收时的标准动作是：抽一条结论 → 要求 `replay --strict` 现场重演。**重演不出来的成果不存在。**
