# 08 · 模拟盘与小资金上线方案

> yquant 阶段二文档 · 编号 08 · v1.0 · 2026-07-05
> 依赖：03 §5.2/§5.5、06（门禁与演练）、07（Gate 证据全部来自事件账本）。
> 立场：上线不是一跃，是**带护栏的阶梯**（ADR-11）。每级只验证它验证得了的东西，判据全部量化、证据全部可回放；同时诚实声明每级**验证不了**什么，防止用"模拟盘跑得不错"欺骗自己。

---

## 1. 五级阶梯总览

| 级 | 名称 | 资金 | 执行方式 | 主要验证对象 | 最短时长 |
|---|---|---|---|---|---|
| L0 | 回测 | 无 | 引擎内 | 策略逻辑与历史约束（03 §8） | — |
| L1 | 影子模式 | 无 | 只记录不执行 | 生产环境下管道稳定性与回放确定性 | ≥20 交易日 |
| L2 | 模拟盘 | 虚拟 ¥300,000 | **系统全自动**（ADR-12） | 全链路精度：撮合/账务/风控/简报/告警 | ≥60 交易日且 ≥2 次月度调仓 |
| L3 | 小资金试点 | 真实小额（§5.3） | 系统建议+**人工确认执行**（ADR-09） | 真实滑点/成本假设、人机流程、预注册区间对照 | ≥6 个月（收益评估 ≥12 个月） |
| L4 | 半自动执行 | — | icebox | 需独立立项与合规评估，本包不涉 | — |

升级只能逐级；任何回退条件触发即降级，降级不丢数据（账本连续）。

## 2. L1 影子模式

策略照常产生 signal/proposal 事件（打 `mode=shadow` 标），无账户、无成交。**出口判据**：连续 20 交易日管道零未处理失败（降级成功计为成功）；周抽 3 日 `replay --strict` 一致率 100%；红旗事件与人工对照巨潮抽查零遗漏。影子期同时完成 06 §6 首次混沌演练。

## 3. L2 模拟盘与 PaperBroker 规格

**PaperBroker（`paper/broker.py`）与回测 ChinaBroker 共享同一约束实现**（同一份 T+1/涨跌停/停牌/费用/整手代码，杜绝两套规则漂移）：

- **订单生命周期**：T 日收盘后管道产出 proposal → L2 下自动转 order（事件入账）→ T+1 开盘撮合：成交价 = 开盘价 ×(1±slippage)，并校验 ∈ 当日 [low, high]，违约束（一字板/停牌/现金不足缩量后仍不足）→ 拒单入账。
- **估值与对账**：日终按收盘价重估，跑 06 §2 P1/P2 对账（分毫必平，不平即 S1 告警并冻结次日自动下单）。
- **股息与除权**：现金分红按到账日入现金；送转按登记日调整持仓；对账须穿越除权日（06 T5 口径）。
- **诚实性约束**：模拟盘的一切成交假设**不得优于**回测假设——由 T7 双引擎等价测试在结构上保证（同策略同区间，回测引擎 vs PaperBroker 逐日推进，净值逐日差 ≤2bps）。
- **数字孪生**：进入 L3 后，L2 模拟账户**不关闭**，作为孪生对照账户与真实账户并行，用于分离"假设误差"与"执行误差"（§5.4）。

## 4. 预注册预期区间书（升 L3 的前置文件）

在看到任何实盘结果**之前**，从 walk-forward 报告生成并签署《预期区间书》，事件入账（不可事后修改，修改即新版本留痕）：

- 组合口径的预期年化收益区间、波动区间、最大回撤上限、月度换手与成本占比预期（取样本外分布的 10–90 分位）；
- 对照跟踪规则：实际值滚动落于区间内 = 正常；跌破下界或回撤超上限 → 触发调查（§5.5），**超出上界同样触发调查**（好得反常往往是错的信号：数据错误、幸存偏差再现、或承担了未识别风险）；
- 明示条款："本区间来自历史模拟，不构成收益承诺；L2 通过仅证明机器可信，不证明策略有效。"

## 5. L2 → L3 升级 Gate 与 L3 运行规则

### 5.1 机器硬门槛（一票否决，证据 = 账本导出包）
① 06 §2 P1–P9 于最近 30 日全绿；② 部署 tag 上 T0–T13 全绿；③ 期内 P0/P1 事故清零且每例已沉淀新测试；④ 红旗遗漏 = 0（周抽查台账）；⑤ 一次消防演习通过（06 §6）；⑥ 抽 3 run 严格回放一致；⑦ 模拟期拒单率与成本占比与回测同期口径差异 <20%（撮合假设自洽性）。

### 5.2 决策门槛（人拍板）
K1 判据复核（03 §9）：若模拟期恰逢策略样本外失效证据增强，允许"机器达标但暂不升级"。升级决定本身作为 human_action 事件入账。

### 5.3 试点资金与仓位纪律
资金规模由委托人书面确定（建议：可投资资产的 ≤5% 且为"全亏不影响生活"的绝对额）；沿用 03 §5.5 风控参数；试点首月新开仓单票 ≤10%（较常规更紧），次月起恢复常规上限。

### 5.4 执行与滑点闭环
人工执行后回填真实成交（价格/股数/费用）→ 系统计算逐笔滑点 = 实际 vs 孪生账户假设，入 `metrics`；月度出《执行质量报告》。**触发线**：滚动 20 笔平均滑点+费用 > 假设值 2 倍 → 暂停新开仓，检修成本模型或执行时点约定（如改为集合竞价执行）。

### 5.5 熔断与回退（触发即执行，解除须人工留痕）
- 组合回撤 ≥10%：禁止加仓（既有规则）；≥15%：**降回 L2**，真实账户只减不增直至评审。
- 单月 ≥2 起 P0 数据/账务事故：暂停新开仓，完成事故五步法后评审恢复。
- 预注册区间跌破下界或上穿上界（滚动季度口径）：启动调查，产出归因报告前维持现状不扩仓。
- 连续 2 周使用频率 <2 次（K3）：冻结并复盘产品形态——试点期同样适用。

### 5.6 L3 收益评估的诚实条款
月频策略 6 个月仅约 6 个独立样本，**试点期的收益数字不足以证明或证伪策略**；12 个月评审时以"是否落在预注册区间 + 执行质量 + 风控纪律执行率"三件套综合判断，而非单看收益率排名。任何"提前加大资金"的冲动都必须走变更流程并重签区间书。

## 6. 运行手册（Runbook，交付物为独立文档，此处定目录）

日检五分钟（数据日期横幅/健康分区/红旗区/待确认 proposal）→ 周检（抽检回放结果、执行质量、LLM 抽检准确率）→ 月度仪式（混沌或消防演练、区间对照回顾、事故与测试沉淀回顾）→ 故障速查表（每条 S1/S2 告警 → runbook 段落 → 具体命令）→ 恢复手册（备份还原、数据勘误流程、模型/prompt 回滚）。

## 7. 验收（对应 WP9 与 WP11）

WP9：PaperBroker 与共享约束层重构完成；T7 全绿；L1 影子 20 日达标报告；L2 首月对账零差异演示；孪生对照报表原型。WP11：运行手册全文、《预期区间书》模板与基于当前 S1–S3 的首份实例、两次演练台账、L2→L3 Gate 评审材料清单（供未来评审直接套用）。
