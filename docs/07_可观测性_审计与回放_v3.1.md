# 07 · 可观测性、审计与回放规范（v3.1）

> 取代旧 07。核心机制**零改动**（事件账本/信封/manifest/回放/五步法均沿用），本版仅做 v3.1 适配。变更点以 ◆ 标记，未标记条款以旧 07 原文为准并视为并入本篇。

## 1. 三层观测体系（不变）
决策事件账本（append-only 唯一事实源）／指标层（06 P1–P11）／日志层（loguru JSON）。**无凭证，不决策**（ADR-13）。

## 2. 来源凭证信封（◆ 增量）
Event.kind 枚举新增：`macro_event_card`、`regime_change`、`committee_output`、`thesis_check`、`overlay_budget_reject`。Provenance 不变（git_sha/config_hash/data_manifest_id/prompt_version/model_id/seed）；◆ `model_id` 记录须含供应商+版本+**knowledge_cutoff**（与 09 联动）。因果链示例更新：`macro_event_card → committee_output(opportunity) → proposal → human_action(confirm+红队已阅) → order → fill → thesis_check…`。

## 3. 数据快照与双时态（◆ 增量）
◆ manifest 覆盖范围扩至信息域全部序列（FRED/指数/汇率/商品/波动/GPR）；宏观序列的官方**修订**（如 NFCI 回溯修订）走勘误分区+新 manifest，旧 manifest 回放不变（T11 覆盖宏观序列）。`asof` 命令支持宏观序列：`yquant asof --date 2026-06-10 --series BAMLH0A0HYM2`。

## 4. 回放引擎与 Run Digest（◆ 微调）
机制不变；◆ 委员会输出（LLM 多步）与所有站点调用一样"录音回放"（原始 request/response 落盘，回放读录音；`--live-llm` 仅用于漂移评估）。◆ 历史事件演习（06 §5）复用同一回放引擎：演习=在 golden manifest 上的正向驱动，其输出同样入账本（打 `drill` 标）以便二次回放演习本身。

## 5. 告警（◆ 微调）
级别与去重规则不变；◆ 新增固定告警源：P10 支柱缺失（S2）、P11 层预算越界（S1）、状态机切换（S3 信息级，Crisis 进入为 S1）。渠道可配置（飞书默认/Telegram/邮件）。

## 6. 事故取证五步法（不变）
冻结→取证→重演→二分定位→归档；每 P0/P1 沉淀 ≥1 条新测试。◆ 二分定位的"模型层"分支增加：对照 knowledge_cutoff 检查是否为污染性回看（ADR-24 场景）。

## 7. 存储与保留（◆ 微调）
金额字段一律 USD；业务时区 America/New_York，存储 UTC ISO8601（T10 保障 DST）；llm_raw 保留 180 天、日志 90 天、账本/manifest/演习台账永久；周冷备+哈希校验（不变）。

## 8. 验收（WP7）
旧 07 §8 全项 + ◆：宏观序列 asof 演示、演习输出的二次回放演示、kind 枚举扩展的信封强校验测试。
