# 01 · 开源竞品调研与生态位分析

> yquant 项目文档包 · 编号 01
> 日期：2026-07-05 · 作者：Claude（受托进行方案完善）
> 目的：回答一个问题——**在 GitHub 已有大量成熟量化/AI 交易项目的前提下，yquant 做什么才不是重复造轮子、才对使用者真正有价值？**
> 注：文中 star 数为调研时点数据，仅用于说明量级，随时间变化。

---

## 1. 调研方法

围绕原方案（v1）涉及的三条能力线——数据/回测、LLM 分析、交易执行——检索并分析了 GitHub 上最具代表性的开源项目，按四个维度评估：

1. **定位**：它声称解决什么问题？
2. **实际边界**：它明确不做什么？（往往藏在 disclaimer 里，最能说明真相）
3. **对 A 股个人投资者的适配度**：数据、市场规则、语言、成本。
4. **对 yquant 的启示**：可以复用什么、必须避开什么、留下了什么空缺。

## 2. 项目逐个分析

### 2.1 microsoft/qlib —— AI 量化研究平台（重研究）

- **定位**：面向 AI 的量化投资平台，覆盖从想法探索到生产的完整链条：数据处理、模型训练、回测、alpha 挖掘、风险建模、组合优化、订单执行；支持监督学习、市场动态建模、强化学习等多种范式，并已接入 RD-Agent 实现研发过程自动化（LLM 自动提出因子/模型假设→写代码→回测→迭代）。
- **实际边界**：本质是**研究基础设施**。数据需要自己准备（官方 A 股数据源时有维护问题）；产出是模型和因子的研究结论，不是一个"每天打开就能用"的个人工具；学习曲线陡峭（自有数据格式、表达式引擎、workflow 配置体系）。
- **A 股个人适配度**：中。支持 A 股数据格式，但个人用户从装好到跑出可信结果的路径很长；对"我该不该动我的持仓"这类日常问题没有直接回答。
- **启示**：
  - ✅ 它验证了"LLM 自动化量化研发"技术上可行（RD-Agent(Q) 论文），但那是研究级玩法，个人跑一轮的算力/API 成本和调试成本都不低。
  - ⚠️ **不要和 Qlib 拼研究平台的深度**——拼不过，也没必要。个人要的不是 SOTA 因子库，是可信的、够用的验证工具。
  - ✅ 可借鉴其对"数据→特征→回测"流水线的抽象，但用轻量方式实现。

### 2.2 virattt/ai-hedge-fund —— LLM 多智能体模拟对冲基金（重演示）

- **定位**：60.8k star 的现象级项目。用 18 个 LLM 智能体模拟传奇投资人（巴菲特/芒格/木头姐/伯里等），各自分析同一只股票后由 Portfolio Manager 智能体汇总出买/卖/持有决策；含回测器和 Web 界面（React + FastAPI + LangGraph）。
- **实际边界**：README 明确声明——**系统不进行任何实际交易，仅供教育与研究目的**。数据依赖付费的 Financial Datasets API，面向美股。近期路线图在向"可回测、可模拟盘、可选实盘"的持续型基金演进，但当前主体仍是决策模拟器。
- **A 股个人适配度**：低。美股数据源、无 A 股交易规则、无中文语料优化。
- **启示**：
  - ✅ 证明了"LLM 人格化多智能体分析"有巨大的**传播力和教育价值**——这是它 6 万星的来源，不是赚钱能力。
  - ⚠️ 关键教训：**多智能体辩论产生的是"看起来专业的意见"，不是经过验证的 alpha**。项目自己都不敢接实盘。yquant 若把 LLM 意见直接当交易信号，等于把演示品当武器。
  - ✅ 它的 Risk Manager 先用确定性规则算出"允许的操作和最大数量"、LLM 只在允许集合内选择的设计（deterministic constraints）非常值得抄——**规则管硬约束，LLM 管软判断**。

### 2.3 TauricResearch/TradingAgents 与 hsliuping/TradingAgents-CN —— 多智能体交易决策框架（重分析，A 股已覆盖）

- **定位**：TradingAgents（56k star 量级）模拟真实交易公司的分工：基本面分析师、情绪分析师、新闻分析师、技术分析师、多空研究员辩论、交易员、风险团队，输出对单只股票的交易建议。TradingAgents-CN（13k+ star）是中文增强版：接入国产大模型（阿里百炼/DeepSeek 等）、A 股/港股数据（Tushare、AkShare、BaoStock 多源）、Streamlit 界面、Docker 一键部署、报告导出 Markdown/Word/PDF。
- **实际边界**：CN 版自我定位写得很清楚——**"学习与研究用途，不提供实盘交易指令"**。单次分析一只股票耗时 2~25 分钟（按研究深度分级），是"按需深度研报生成器"，不是每日自动化投研流；无组合管理、无交易纪律闭环、无回测验证其建议的历史胜率。维护上长期单人开发，作者公开求测试志愿者，工程质量波动。
- **A 股个人适配度**：中高（这是离 yquant 最近的竞品）。
- **启示**：
  - ⚠️ **这是必须正面回答的问题：TradingAgents-CN 已存在，yquant 为什么还要做？** 答案见 §3 空缺分析——它是"单票深度研报生成器"，缺的是：①对建议质量的**回测验证**（它的建议历史上到底准不准？没人知道）；②**组合级**日常工作流（我的 20 只自选股今天发生了什么）；③**交易纪律与复盘闭环**（建议→执行→记录→归因）。
  - ✅ 多 LLM 适配器、报告导出、Docker 部署等工程实践可直接参考。
  - ✅ 它验证了国产 LLM（DeepSeek/千问）做中文财经文本分析的成本可行性。

### 2.4 UFund-Me/Qbot —— AI 自动量化机器人（大而全）

- **定位**：宣称"AI 自动量化交易机器人，完全本地部署"，覆盖数据、策略、回测、模拟盘到实盘的全流程。
- **实际边界**：典型的"什么都有一点"的项目，模块间集成度和文档质量参差，社区反馈中"跑通成本高"是常见抱怨；实盘部分依赖的第三方接口稳定性存疑。
- **启示**：⚠️ **反面教材式的警示：大而全 = 每个环节都半成品**。yquant v1 范围必须收窄到"少数几件事做到可信"。

### 2.5 回测引擎层：backtrader / vectorbt / zipline-reloaded

- **backtrader**：最流行的 Python 事件驱动回测框架，文档全、中文社区大、支持滑点/佣金模拟；缺点是 2023 年后原作者基本停更（社区分支维护）、速度慢、对 A 股规则（T+1、涨跌停、停牌）**没有内置支持，需要自己实现约束层**。
- **vectorbt**：向量化回测，速度快几个数量级，适合大规模参数扫描；开源版功能受限（Pro 收费），学习曲线陡，事件级细节（逐笔撮合规则）表达力弱。
- **zipline-reloaded**：Quantopian 遗产的社区维护版，美股基因重，A 股适配成本高。
- **启示**：✅ 选 backtrader 作为引擎底座是合理的（原方案 v1 判断正确），但**必须自建一层 "A 股约束适配器 + 回测可信度测试集"**，这正是几乎所有开源项目都没做好的部分，也是本项目回测结果可信的前提。

### 2.6 数据层：AkShare / Tushare / BaoStock

- **AkShare**（21k star）：免费、接口面广（行情/公告/新闻/龙虎榜/北向），但本质是对东财/新浪等网页接口的封装，**接口随上游改版而失效是常态**，字段口径偶有变动。
- **Tushare Pro**：数据规范、文档好，但核心接口需要积分（约每年 200~500 元档位才够用），公告全文类数据仍有限。
- **BaoStock**：免费稳定但覆盖窄（主要是行情与部分财务）。
- **启示**：✅ **数据源必须做适配器模式 + 本地落盘缓存**，任何上游接口都视为"随时会挂"，落盘后的本地数据才是系统的真实数据源。这一条要作为硬性架构约束写入方案。

### 2.7 交易执行层：vnpy / easytrader / QMT(xtquant)

- **vnpy**：成熟的实盘交易框架，接口丰富，但以期货/期权为主力场景，A 股个人接入仍绕不开券商侧门槛。
- **QMT/xtquant**：券商官方量化终端，A 股个人程序化下单的主流合规通道，但需要满足券商开通条件（通常有资金门槛），且 2024–2025 年程序化交易监管新规后，自动化下单的报备与合规要求趋严。
- **启示**：⚠️ **v1 阶段不做自动下单**不只是求稳，也是合规上的明智选择。人工下单 + 系统记录，已能覆盖个人低频策略的全部需求。

### 2.8 在线平台参照：聚宽 / BigQuant / 果仁网

不开源，但它们定义了"个人量化"的用户体验基线：网页写策略、平台数据、一键回测。弱点：策略代码在平台上（隐私/迁移风险）、LLM 能力弱或收费、无法与自己的真实持仓和交易记录打通。yquant 的"本地化 + 私有数据 + AI"是相对这些平台的差异点。

## 3. 空缺分析：现有生态没有覆盖的位置

把上述项目按两个轴摆开——**横轴：分析深度（规则/统计 ←→ LLM 深度研判）；纵轴：离真金白银的距离（教育演示 ←→ 实盘辅助）**：

```
                     离实盘近（管真钱）
                          ▲
          vnpy/QMT ●      │
        （只管执行，       │      ★ 空缺区：
          不管研究）       │      「个人日常投研工作台
                          │        + 纪律化执行辅助」
                          │        ← yquant 目标位置
   ───────────────────────┼──────────────────────────▶
   规则/统计              │              LLM 深度研判
                          │
      backtrader ●        │        ● TradingAgents-CN
      vectorbt ●          │        （单票深度研报，
      Qlib ●              │          学习研究用途）
    （研究基础设施）        │        ● ai-hedge-fund
                          │        （教育演示，不实盘）
                          ▼
                     离实盘远（教育/研究）
```

**空缺区的具体含义**——市面上没有一个开源项目同时做到以下四点：

1. **组合视角的每日自动投研流**：不是"我输入一只股票你分析 20 分钟"，而是"每天收盘后，我的自选股池发生的所有重要事情（公告、异动、财务变化）被自动读完、分类、排序，5 分钟看完该看的"。
2. **LLM 输出可验证、可追溯**：每条 AI 判断附原文链接与依据，事件分类有标注集验收，禁止编造数字——把 LLM 当"不知疲倦的初级研究员"用，而不是当"预言家"用。
3. **回测服务于证伪而非寻宝**：内置 A 股真实约束（T+1/涨跌停/停牌/退市股/真实费率）与"回测陷阱测试集"，先保证"跑出来的数字可信"，再谈策略好坏。
4. **交易纪律闭环**：建议→人工确认→执行记录→AI 复盘归因，针对个人投资者最大的亏损来源——行为偏差——做工程化约束。

## 4. 结论：三条战略判断

**判断一：不做第 N 个"AI 预测/多智能体荐股"项目。** 该赛道已有 6 万星和 5.6 万星的头部项目，且它们全都不敢宣称能实盘赚钱——这本身就是最强的市场信号。

**判断二：yquant 的生态位 = "A 股个人投资者的 AI 投研副驾 + 纪律引擎"。** 复用成熟组件（AkShare/backtrader/Streamlit/国产 LLM），把工程精力集中在四个无人认真做的点上：数据可靠性、回测可信度、LLM 可验证性、纪律闭环。

**判断三：能复用的绝不自研。** 具体复用清单与自研边界见《03 技术方案 v2》§4。
