## 1. 本包缘起与设计立场

委托人新目标：**系统先在模拟盘上跑，表现可以则以小资金实盘试点**；并要求——高精度、稳定运行、无论成功还是犯错都能收集足够信息**还原现场**、可回测、可解释；即便引入 NN 这类黑箱模型，系统也必须能说清"这个黑箱表现到底如何"。

我对这个目标做了一次关键的诚实校准，它决定了整个包的设计：

**"模拟盘表现可以"必须拆成两件事。** 第一件是**机器可信**：数据对不对、账算得准不准、故障扛不扛得住、每个决策能不能逐字节还原——这件事 60 个交易日足够验证，且必须 100% 达标。第二件是**策略赚钱**：月频策略 60 个交易日只有约 3 次独立调仓样本，**在统计上不可能被证实或证伪**；谁声称"模拟盘两个月证明了策略有效"谁就在自欺。所以本包把上线判据设计为：**机器指标做硬门槛（全部量化、一票否决），收益指标做"预注册区间对照"（回测给出预期区间，实盘只检验是否落在区间内，偏离即调查）**——这既满足"小资金试一试"的诉求，又不给系统制造说谎的机会。

## 3. 对阶段一文档的修订（以本包为准）

- 03 §2.4 非目标中"不做自动下单"细化为：**真实资金的自动下单仍不做**（ADR-09 不变）；**模拟账户内的自动执行纳入范围**（这正是模拟盘的意义，见 ADR-12）。
- 03 §5.7 调度、§7 存储：按 07 篇扩展（decision_events、data_manifests 等新表与 run digest 机制）。
- 03 §8 陷阱测试集：按 06 篇扩展 T7–T12（v3 进一步扩至 **T7–T15**，新增 T14 波动率目标器/T15 拥挤流动性哨兵，见下方 v3 修订点与 06 §3）。
- 03 §9 里程碑：按 10 篇在 WP2 之后插入 WP7（回放地基），原 M4–M6 顺延（v3 另在里程碑序列末尾插入 **M8 风险引擎**，见 03 §9/§5.8、10 篇 WP12）。
- 02 篇 ADR 追加如下：

| 编号 | 决策 | 一句话理由 |
|---|---|---|
| ADR-11 | 确立五级上线阶梯 L0 回测→L1 影子→L2 模拟盘→L3 小资金试点→L4 半自动执行；L4 保持 icebox，需独立决策 | 把"试一试"变成有护栏的阶梯而非一跃 |
| ADR-12 | 模拟账户内允许全自动执行；真实资金执行维持"系统建议+人工确认"，ADR-09 不变 | 模拟盘验证全链路必须自动；实盘自动化的风险/合规结论未变 |
| ADR-13 | 全系统强制"事件溯源 + 数据快照清单"架构约束，原则为**无凭证，不决策**：任何进入账本的决策必须携带完整来源凭证并可回放 | "还原能力"不能靠事后打捞日志，必须是架构性质 |
| ADR-14 | 黑箱模型（NN 等）允许作为候选信号源，但必须通过 09 篇治理阶梯准入，且每个信号强制携带"解释契约"对象 | 不禁止黑箱，禁止不可度量、不可监控的黑箱 |
| ADR-15 | "高精度"落地为 06 篇 §2 精度指标表并纳入 CI；任何指标降级视为发布阻断 | 形容词不可验收，数字可以 |

### v3 修订点（依据 12 篇 SSOT，本包相关篇已落地）

- **目标函数（12 §1）**：上线判据的"收益指标"侧统一收敛为**风险调整收益（卡玛=年化收益/最大回撤，辅夏普与月度亏损尾部）**；预注册区间书（08 §4）据此按核心/卫星分层给区间。交易成本与滑点照常建模（损耗，非税），仅**税务建模与 LLM 日预算硬上限冻结**（`tax_*` 默认置零、用量记账保留，ADR-27）。
- **组合架构（12 §4，ADR-23）**：确立**核心–卫星–对冲**；09 篇 provider 治理、08 篇预注册分层、06 篇测试均按核心 C1–C3 / 卫星 S-A/S-B/S-C / 对冲的分层口径落地。
- **事前仓位控制 M8（12 §5，ADR-25/26）**：新增波动率目标器 + 趋势闸门 + 拥挤/流动性哨兵 + 熔断阶梯；06 §2 增 P10 触发正确性、§3 增 T14/T15，08 §5.5 熔断并入 M8 第④条，10 篇新增 WP12 交付 `risk/` 模块与 `risk_events` 表。
- **LLM/ML 松绑（12 §3 J2，ADR-22）**：LLM/ML 可作 SignalProvider **产评分不产订单**（推翻 ADR-03 禁令部分、保留不下单内核），走 09 篇治理阶梯 + 人工确认执行。
- **污染约束 J3（12 §3、§6，ADR-24）**：`kind∈{llm,ml_blackbox}` 的 provider 评估**仅承认 ModelCard `knowledge_cutoff` 之后样本**，截止前标 `contaminated` 仅供参考；ModelCard 必填 `knowledge_cutoff`，缺失拒准入（09 §2、06 §7 CI 断言落地）。
- **ADR-22~28**：v3 追加决策全部登记于 02 篇 ADR 索引（LLM 产评分/核心–卫星–对冲/知识截止/M8/拥挤哨兵/税务与预算冻结/A 股经 ETF·港股 icebox）。
- **文档缺口**：12 §7 引用的 **11 篇（港股与税务复杂度 R2/R3/R6/R7）在当前 `docs/` 中缺失**（现存 00–10、12）；涉及港股 icebox 与税务冻结的详细依据待补该篇，凡引用处如实标注"待补 11 篇"，不据缺失内容杜撰。
